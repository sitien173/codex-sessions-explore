# Codex Session Explorer

A fast, beautiful developer tool for browsing and inspecting recorded [Codex](https://github.com/openai/codex) coding agent sessions stored as JSONL archives.

![Session Explorer](docs/screenshots/explorer.png)

## Features

- **Session Grid** — browse all sessions with project, date, model, git branch, and file size at a glance
- **Instant Search** — full-text search via [MiniSearch](https://github.com/lucaosti/minisearch) with fuzzy matching and prefix support (<5ms latency)
- **Project Filtering** — filter sessions by project (derived from the working directory)
- **Sort Controls** — sort by newest first or largest file
- **Session Timeline Viewer** — opens any session on demand and renders a full event timeline:
  - 🔵 **User messages** — your prompts to the agent
  - 🟢 **Assistant messages** — agent responses
  - 🟣 **Agent reasoning** — collapsible thinking blocks
  - 🟠 **Tool calls** — `exec_command`, patches, and their outputs
- **No database required** — runs entirely on static JSON index files and on-demand JSONL fetching
- **Streaming indexer** — processes 700 KB+ files in milliseconds by stopping after the first user message

## Tech Stack

| Layer | Technology |

## Getting Started

### 1. Install dependencies

```bash
npm install
```

### 2. Add your session files

Place your JSONL session archives in the `sessions/` directory, organized as `YYYY/MM/DD/`:

```
sessions/
  2026/
    03/
      06/
        rollout-2026-03-06T03-03-45-019cc119-....jsonl
```

> **Tip:** If your sessions live elsewhere, pass the path directly to the indexer (see below).

### 3. Build the index

```bash
npm run build-index
```

This streams every JSONL file, extracts the `session_meta` and the first `user_message`, and writes:

- `public/sessions.json` — metadata for all sessions (~100–200 KB for 260 sessions)
- `public/search_index.json` — text snippets for MiniSearch

#### Custom sessions directory

```bash
npx tsx scripts/build-index.ts --sessions-dir /path/to/your/sessions
```

### 4. Start the dev server

```bash
npm run dev
```

Open [http://localhost:5173](http://localhost:5173).

### 5. Production build

```bash
npm run build
```

Output in `dist/`. Deploy as any static site (Nginx, Caddy, GitHub Pages, Vercel, etc.).

## JSONL Session Format

Each session file starts with a `session_meta` event on line 1, followed by a mix of event types:

```jsonl
{"type":"session_meta","payload":{"id":"019cc119-...","timestamp":"2026-03-06T03:03:45.060Z","cwd":"/opt/my-project","model_provider":"openai","cli_version":"0.98.0","git":{"branch":"main","repository_url":"https://github.com/..."}}}
{"type":"response_item","payload":{"type":"message","role":"user","content":[{"type":"input_text","text":"..."}]}}
{"type":"event_msg","payload":{"type":"user_message","message":"generate docker compose file"}}
{"type":"event_msg","payload":{"type":"agent_reasoning","text":"Planning the approach..."}}
{"type":"event_msg","payload":{"type":"agent_message","message":"I'll generate a docker-compose.yml..."}}
{"type":"response_item","payload":{"type":"function_call","name":"exec_command","arguments":"{\"cmd\":\"ls\"}"}}
{"type":"response_item","payload":{"type":"function_call_output","output":"..."}}
```

| Event type | Description |
|---|---|
| `session_meta` | Session ID, timestamp, cwd, model, git info |
| `event_msg` / `user_message` | Your prompt to the agent |
| `event_msg` / `agent_message` | Agent's text response |
| `event_msg` / `agent_reasoning` | Agent's internal thinking (collapsible) |
| `response_item` / `function_call` | Tool invocation (exec_command, patch, etc.) |
| `response_item` / `function_call_output` | Tool result |

## Performance

- **Initial load:** ~100–200 KB total for 260 sessions (sessions.json + search_index.json)
- **Search latency:** <5ms for any query via MiniSearch in-memory index
- **Session detail:** single JSONL file fetched on demand (only when you open a session)
- **Indexer:** stops reading each file after the first `user_message` — no full-file parsing
